Interview · 六題訪談

Six Questions
№ 01

為什麼從行政與電子維修，轉身做前端？

做行政與維修這幾年，我發現自己最有成就感的時刻，都是「把混亂的事情整理成有秩序的介面」—— 無論是排報表、修硬體還是畫 AutoCAD。 前端剛好把這件事放大成一個職業， 能用程式碼放大十倍的影響力。所以我認真學了。

№ 02

主力是 Vue 還是 Angular？兩個都會嗎？

Vue 3 是我最常用的，Vite + Pinia + vee-validate 完整側專案都跑過； Angular 19 是團隊專案 LazyBaoBao 的主力框架， 我擔任前端負責人，Standalone Components、Signals、RxJS Reactive Forms 全部實戰過。 兩個都會，不偏食。

№ 03

作品集中最有挑戰的是哪一個？

LazyBaoBao。一個系統要服務客戶、POS 終端、老闆後台三個角色， 狀態同步、權限分離、UI 一致性都要兼顧； 還做了 3D 翻卡支付、Lottie 動畫、Mock 模式離線測試。 學到最多的是「大型專案怎麼拆模組、讓多人協作不打架」。

№ 04

沒有資工背景，會不會是劣勢？

電子科出身、做過硬體維修，對底層怎麼跑的有直覺； 做過 AutoCAD / SketchUp 製圖，對版面、間距、比例敏感； 做過行政，知道怎麼把需求文件拆成可執行步驟。 這些反而是跨域整合的優勢。

№ 05

期望的工作型態？

高雄上班或遠端皆可，正職／半天皆考慮。 最在意的是有人能 code review、能學東西的環境——比起薪水，我更想要成長空間。

№ 06

下一步想學什麼？

短期把 React + Next.js 補起來，三大框架都有實戰； 中期深入 TypeScript 進階型別、效能調校與 a11y 無障礙； 長期目標是能獨立負責一條產品線的資深前端。
